⏺ This is the browser sidecar container image for OpenClaw. I changed it from browserless/chrome:latest to chromedp/headless-shell:latest.

  What it does: It runs a headless Chromium browser accessible via the Chrome DevTools Protocol (CDP). The OpenClaw agent uses this to browse the web — it connects to the
  browser container over the Docker network on port 9222 (chromedp/headless-shell's default CDP port, versus browserless which used port 3000).

  Why the change: browserless/chrome is a heavier, commercial image with a license/token system. chromedp/headless-shell is a minimal, open-source headless Chrome image
  maintained by the chromedp Go project. It's smaller, simpler, and has no licensing overhead — just a bare headless Chromium exposing CDP. This aligns with the "subtract
  don't add" principle: we don't need browserless's extra features (recording, debugging UI, queue management), we just need a headless browser the agent can drive.

  How it works: The container sits on the same Docker network as the OpenClaw container (Coolify's network). The agent reaches it at browser:9222 (or whatever port is
  mapped) via CDP. The shm_size: "1gb" is there because Chromium uses /dev/shm for shared memory and the default 64MB Docker allocation causes tab crashes on heavier pages.
